Abstract

We introduce tulip agent, an architecture for autonomous agents with access to vast amounts of tools via a tool library. In contrast to state-of-the-art implementations, descriptions of available tools are not encoded into the system prompt, as this counts against the model's context window, may lead to needle-in-the-haystack issues for large numbers of tools, and results in static capabilities. Instead, the tulip agent can search for suitable tools in its extensible tool library, exemplarily implemented as a vector store. The architecture is evaluated regarding scalability using mathematical tasks and tools, and its generalizability is demonstrated via an application to robotics.

System

The tulip agent architecture includes several key components: the language model, the tool library, the tools themselves, a search module, and the function execution module.

The tulip agent architecture.

Evaluation

Preliminary evaluation results on math tasks hint towards the following insights: 1) chain-of-thought reasoning significantly improves the results, 2) tools are a prerequisite for getting the final responses right, and 3) the tulip agent architecture significantly reduces costs, and the CotTulipAgent maintains correctness.
